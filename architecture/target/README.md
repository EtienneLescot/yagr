# Target Architecture

Cette section est ephemere.

Elle sert a documenter:

- l'architecture cible
- les ecarts avec l'existant
- le backlog de convergence

## Regle de vie

Quand une cible est implementee:

1. la documentation descriptive doit etre deplacee ou consolidee dans `../current/`
2. l'item correspondant doit etre retire de `refactor-backlog.md`
3. le contenu obsolete doit etre supprime, pas archive indefiniment

## Contenu

- `architecture-target.md`: architecture souhaitee
- `refactor-backlog.md`: plan de convergence restant
- `provider-capability-implementation-plan.md`: decisions validees et plan d'implementation sur la couche providers/capacites
